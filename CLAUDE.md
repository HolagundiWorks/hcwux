# hcwux — agent instructions

**This repository is the CANONICAL SOURCE of the HCW design system** — `@hcw/ui-kit`
(HCW-UI-Kit) — plus its governance and the AI-Orchestration UX framework. Consumer
apps (AORMS `esti`, and siblings) vendor the **built** output of this repo; they do
**not** own the kit. **Edit the kit here, in `src/`.**

## Layout

| Path | What it is |
| --- | --- |
| `src/` | The kit source (TypeScript + `portal-chrome.scss`). **The source of truth.** |
| `docs/styleguide.html` | Self-contained **visual styleguide** (foundations · layers · components · elements · resources). Keep its token values in sync with `src/tokens.ts` when tokens change. |
| `dist/` | **Built output — gitignored. Never commit it.** Generated by `pnpm build` (tsc → JS + `.d.ts`, copies scss). |
| `package.json` | The npm package (`@hcw/ui-kit`) at the repo root. `main`/`exports` point at `dist/`. |
| `docs/hcw-kit/` | Governance tree 00–13 (Constitution, token governance, component quality, templates, MUI mapping, audits, AI-agent rulebook). |
| `docs/esti/HCW-*.md` | Layers/spatial model, UX principles, AI knowledge base, AI-Orchestration UX doctrine. |
| `docs/ai-orchestration-ux-framework.md` · `docs/esti/HCW-AI-ORCHESTRATION-UX.md` | AI-orchestration doctrine · ESTI/AORMS application (HCW kit primitives). |
| `docs/ADOPTING-THE-KIT.md` | How consumers (esti) vendor the built kit. |
| `.github/workflows/ci.yml` | install(prepare→build) · typecheck · test · verify dist · **size budget**. |

## Updating the kit — the workflow

1. **Edit `src/`** in the redesign order: `src/tokens.ts` → `src/chrome-sx.ts` →
   the component → `src/portal-chrome.scss`. Shared visual changes land here **once**;
   they are never forked into a consuming app.
2. **`pnpm build`** — regenerates `dist/`.
3. **`pnpm typecheck` + `pnpm test`** must be clean (51+ tests), and **`pnpm size`**
   must stay under the gzip budget.
4. **Bump `package.json` `version` + add a `CHANGELOG.md` entry** for any
   design-contract change (semver — see `GOVERNANCE.md`).
5. **Commit `src` + `package.json` + `CHANGELOG.md` (and any docs) and push.**
   **Do NOT commit `dist/`** — it is gitignored and built by CI/consumers.

## Consumers — re-vendoring after a change

Apps consume the **built** kit, not this source. For AORMS `esti`, the kit is vendored
at `esti/vendor/hcw-ui-kit/` (esti's Docker build is self-contained and has **no git**,
so pnpm git-dependencies are not usable there). After a kit change lands here:

- In `esti`: copy **this repo's fresh `dist/`** → `esti/vendor/hcw-ui-kit/dist/`,
  `git add -f` (esti's `.gitignore` has a global `dist/` rule; drop `*.map`), bump
  `esti/vendor/hcw-ui-kit/package.json` `version`, and commit.

Full detail: `docs/ADOPTING-THE-KIT.md`.

## Non-negotiables

- **Human Centric _Works_, not Design (HCD).** The kit is judged by whether the design
  *works* under pressure — functional, legible, operational — not by aesthetics.
  Aesthetics serve function; when taste and UX law conflict, **UX law wins.**
- **Never commit `dist/`** — it is built output.
- **Thesis: depth encodes importance** — flat (info) · soft (objects) · glass
  (action). Square surfaces (`RADIUS` 0); only buttons/dialogs/dock rounded. **One**
  accent — Radiant Orange `#FF4F18` (fills/CTAs only; links slate).
- **Keep the kit domain-agnostic** — no app/domain concepts leak into `src/`.
- Every component: token-driven (no raw hex outside `tokens.ts`), exported prop types,
  unit-tested, `:focus-visible` = hover-glass parity. WCAG 2.2 AA is a gate.
- Brand font: **Urbanist**. MUI is implementation only; HCW Kit is the single source of truth.
