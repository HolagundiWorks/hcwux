# HCW-UI white papers

The philosophy underneath the design system — *why* the interface is the way it is,
before any token or component. Read these to understand the system; read the
governance docs to enforce it.

| # | Paper | What it argues |
| --- | --- | --- |
| 01 | [**Depth Encodes Importance**](depth-encodes-importance.md) | The three material languages — **flat → soft → glass** — as a scale of demand. Why the z-axis carries meaning, why glass is scarce, and why most of the screen stays calm and flat. |
| 02 | [**The Calm Interface**](the-calm-interface.md) | The operating philosophy: design for the operator under pressure, not the explorer. Chrome serves the task, one spatial model, disclose the next step, work without a mouse. |
| 03 | [**AI-Orchestration UX**](../ai-orchestration-ux-framework.md) | Mission-first, not conversation-first. The human supervises intent; the AI manages execution. How the same philosophy governs AI surfaces. ([interactive prototype](../ai-orchestration-ux-prototype.html)) |
